Git Versus Github

  • Git: version-control software
    • Lets you keep track of source code history
  • Github: web-based Git version control repository hosting service
    • Helps manage open-source projects that use Git

Git Flow Model

Our own project!

Now we'll set up our own Github project

Let's start by creating a github account.

Go to www.github.com and click "Sign Up" in the top right.

Now we'll create a copy (or "fork") of the Github project.

  • Go to https://github.com/Affirm/affirm-cares-github-workshop.
  • Click the "Fork" button in the top right.

If you see a popup asking you where to fork it, click your username.

After a minute or so, you should be taken to your new forked repository (repo).

Click on the "index.html" file in your new repo.

Click on the pencil icon to edit the file.

My favorite hobby is watching and playing sports.

When you're finished, add a title and description for your change.

Select "Create a new branch and start a pull request".

On the next page, you'll be able to see the code you've changed by scrolling to the bottom.

If it looks right, click "Create pull request".

Normally, someone would now review your pull request.

For now, let's just click "Merge pull request", followed by "Confirm merge".

Now, the pull request should appear as "Merged" in purple.

Let's go back to the repository main page by clicking "Code"

Let's click on the "index.html" file again.

You can now see your change!
